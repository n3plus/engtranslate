시작하기 Git의 짧은 역사
A Short History of Git
Git의 짧은 역사

As with many great things in life, Git began with a bit of creative destruction and flery controversy.
인생의 많은 위대한 일들과 마찬가지로 Git은 약간의 창조적 파괴와 뜨거운 논쟁으로 시작되었습니다. 

The Linux kernel is an open source software project of fairly large scope.
Linux 커널은 상당히 넓은 범위의 오픈소스 소프트웨어 프로젝트 입니다. 

During the early years of the Linux kernel maintenance (1991-2002), changes to software were passed around as patches and archived files.
Linux 커널 유지관리 초기동안 소프트웨어 변경 사항은 패치와 아카이브 파일로 전달되었습니다. 

In 2002, the linux kernel project began using a proprietary DVCS called BitKeeper.
2002년에 Linux커널 프로젝트는 BitKeeper라는 독점 DVCS를 사용하기 시작했습니다. 

In 2005, the relationship between the community that developd the Linux kernel and the commercial company that developed BitKeeper broke down, and the tool’s free-of-charge status was revoked.
2005년에는 Linux 커널을 개발한 커뮤니티와 BitKeeper를 개발한 사업회사간의 간계가 깨졌고, 도구의 무료 상태가 취소되었습니다. 

This prompted the Linux development community (and in particular Linus Torvalds, the creator of Linux) to develop their own tool based on some of the lessons they learned while using BitKeeper.
이로인해 Linux 개발 커뮤니티(특히 Linux 창시자인 Linus torvalds)는 BitKeeper 를 사용하면서 배운 몇가지 교훈을 바탕으로 자체 도구를 개발하게 되었습니다. 

Some of the goals of the new system were as follows
새로운 시스템의 목표 중 일부는 다음과 같습니다. 

Speed / simple design / Strong support for non-linear development(thousands of parallel branches) / fully distributed / able to handle large project like the Linux kernel efficiently (speed and data size)
속도 / 심플한 디자인 / 비선형 개발에 대한 강력한 지원(수천개의 병렬 분기) / 완전분산 /  Linux 커널과 같은 대규모 프로젝트를 효율적으로 처리할 수 있음(속도 및 데이터 크기)

Since its birth in 2005, Git has evolved and matured to be easy to use and yet retain these initial qualities.
2005년에 탄생한 이후 Git은 사용하기 쉽도록 진화하고 성숙해 가면서 이러한 초기 특성을 유지합니다. 

It’s amazingly fast, It’s very efficient with large projects, and it has an incredible branching system for non-linear development
놀롭도록 빠르고, 대규모 프로젝트에서 매우 효율적이며 비선형 개발을 위한 놀라운 분기 시스템을 갖추고 있습니다.
